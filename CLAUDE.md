## Project Overview

A [Red Discord Bot](https://docs.discord.red/) cog that integrates with the [Pelican Platform API](https://docs.pelicanplatform.org/api-docs) for server administration. The bot runs inside a Pelican Python 3.11 container and exposes Discord commands for a group of friends to manage their server infrastructure.

## Red Bot Cog Structure

Every Red cog follows this file layout:

```
Pelican_COG/
├── __init__.py       # Cog discovery — imports cog class and defines async setup(bot)
├── pelican.py        # Main Cog class
├── info.json         # Cog metadata (name, description, author, python_requires, etc.)
└── requirements.txt  # Extra pip deps beyond what Red provides
```

### Minimal `__init__.py`
```python
from .pelican import PelicanCog

async def setup(bot):
    await bot.add_cog(PelicanCog(bot))
```

### Minimal Cog skeleton
```python
from redbot.core import commands, Config, checks
import aiohttp

class PelicanCog(commands.Cog):
    """Pelican server administration"""

    def __init__(self, bot):
        self.bot = bot
        self.config = Config.get_conf(self, identifier=<unique_int>, force_registration=True)
        self.config.register_global(pelican_url="", api_token="")
        self.session = aiohttp.ClientSession()

    async def cog_unload(self):
        await self.session.close()
```

## Key Red Bot Patterns

### Config (persistent storage)
```python
# Register defaults in __init__
self.config.register_global(key=default_value)
self.config.register_guild(key=default_value)   # per-server settings

# Read / write
value = await self.config.key()
await self.config.key.set(new_value)

# Guild-scoped
value = await self.config.guild(ctx.guild).key()
await self.config.guild(ctx.guild).key.set(new_value)
```

### Permission checks (decorators)
```python
from redbot.core import checks

@checks.is_owner()        # bot owner only
@checks.admin_or_permissions(administrator=True)
@checks.mod_or_permissions(manage_guild=True)
```

### Command types
```python
@commands.command()                          # prefix command  (!pelican status)
@commands.group()                            # prefix group    (!pelican ...)
@commands.hybrid_command()                   # prefix + slash
@commands.hybrid_group()                     # prefix + slash group
```

### HTTP calls (use the shared session, never requests)
```python
async with self.session.get(url, headers=headers) as resp:
    data = await resp.json()
```

## Pelican API Integration

**Pelican Panel** (pelican.dev) is a Laravel-based game server management panel (Pterodactyl fork).

**Integrated API docs** are served live at:
- `https://<your-panel>/api/application` — admin API (OpenAPI/Scramble UI)
- `https://<your-panel>/api/client` — client API (OpenAPI/Scramble UI)

**Auth:** API key in header — `Authorization: Bearer <api_key>`
**Format:** JSON
**Config storage:** `pelican_url` (panel base URL) and `api_token` in `self.config`

### Client API (`/api/client`) — actions on accessible servers

| Endpoint | Method | Purpose |
|---|---|---|
| `/api/client/account` | GET | View own account |
| `/api/client/account/api-keys` | GET/POST | List / create API keys |
**Important:** Client API uses the short `identifier` (8-char, e.g. `8551552c`), NOT the full UUID.

| Endpoint | Method | Purpose |
|---|---|---|
| `/api/client` | GET | List all accessible servers |
| `/api/client/servers/{identifier}` | GET | View server details |
| `/api/client/servers/{identifier}/resources` | GET | CPU/RAM/disk/network utilization |
| `/api/client/servers/{identifier}/power` | POST | Power action (`start`/`stop`/`restart`/`kill`) |
| `/api/client/servers/{identifier}/command` | POST | Send console command |
| `/api/client/servers/{identifier}/backups` | GET/POST | List / create backups |
| `/api/client/servers/{identifier}/files` | GET | List files |
| `/api/client/servers/{identifier}/activity` | GET | Activity log |

### Application API (`/api/application`) — admin management (requires admin API key)

**Note:** Returns 404 if the API key used is a client-scoped key. Requires a separate application API key generated by a panel admin.

| Endpoint | Method | Purpose |
|---|---|---|
| `/api/application/servers` | GET/POST | List / create servers |
| `/api/application/servers/{id}/suspend` | POST | Suspend server |
| `/api/application/servers/{id}/unsuspend` | POST | Unsuspend server |
| `/api/application/servers/{id}/reinstall` | POST | Reinstall server |
| `/api/application/users` | GET/POST | List / create users |
| `/api/application/nodes` | GET/POST | List / create nodes |

### Power action body
```json
{ "signal": "start" }   // start | stop | restart | kill
```

### Helper pattern
```python
async def _api_get(self, endpoint: str) -> dict:
    base = (await self.config.pelican_url()).rstrip("/")
    token = await self.config.api_token()
    headers = {"Authorization": f"Bearer {token}", "Accept": "application/json"}
    async with self.session.get(f"{base}{endpoint}", headers=headers) as resp:
        resp.raise_for_status()
        return await resp.json()
```

## Development Notes

- **Python version:** 3.11 (matches the Pelican container)
- **No external HTTP library needed** — `aiohttp` ships with Red
- **Always close the aiohttp session** in `cog_unload`
- **Config identifier** must be a unique integer per cog — pick a large random one and never reuse it
- **`info.json` `end_user_data_statement`** is required by Red 3.5+ for cog store compatibility

## Deployment

### Folder layout

Red requires cogs to be packages inside a **parent folder**. You add the parent with `[p]addpath`, then load the cog by its folder name.

```
<your-dev-folder>\          ← [p]addpath points here
└── Pelican_COG\            ← cog package folder ([p]load Pelican_COG)
    ├── __init__.py
    ├── pelican.py
    └── info.json
```

### Loading on the Red bot container

The Red bot runs inside a Pelican Panel container. Transfer the `Pelican_COG` folder to the container via SFTP (host and port found in the Pelican panel under your server's settings).

The Pelican file manager shows paths relative to `/home/container/`. The cog lives at:
```
/home/container/cogs/Pelican_COG/
```

In Discord:
```
[p]addpath /home/container/cogs
[p]load Pelican_COG
[p]pelican setup https://<your-panel-url> <api-key>
```

### Reloading after edits

```
[p]reload Pelican_COG
```

### Local dev (Windows)

```
[p]addpath C:\path\to\your\dev\folder
[p]load Pelican_COG
[p]reload Pelican_COG
```
